# Mtn — Figma Wireframe Procedure (Simplified)

This procedure gets you from zero to a wireframe Claude can analyze. It takes about 60–90 minutes and requires no design experience. You are not designing a finished product — you are arranging placeholder blocks to communicate screen structure.

---

## Step 1 — Get a Template (10 min)

1. Go to [figma.com/community](https://figma.com/community)
2. Search: `dashboard wireframe kit`
3. Look for a result with a high like count (1000+) and a preview that shows:
   - A sidebar or top navigation
   - Cards with numbers or stats
   - Some kind of list or feed
4. Good starting points to look for by name:
   - **"Wireframe Kit"** by Figma (official)
   - **"Untitled UI Wireframe Kit"**
   - Any kit labeled "dashboard" or "admin" in grayscale
5. Click **Duplicate** or **Open in Figma** to copy it to your drafts
6. Rename the file: `Mtn — Wireframes`

You are not going to use most of what's in this kit. You are using it as a parts library.

---

## Step 2 — Set Up 4 Blank Screens (5 min)

Create 4 frames (use `F` shortcut in Figma, select **Desktop → 1440**):

Label them exactly:
1. `Morning Briefing` — the main screen, opened daily
2. `Log Training Session` — a form to record a workout
3. `Set Target Adventure` — a form/view to set the goal trail
4. `Meal Plan` — displays the current 3-day meal plan

These are the only screens Cycle 0 and Cycle 1 need. Don't add more.

---

## Step 3 — Fill Each Screen (40–60 min)

Copy components from the kit and paste them into each frame. Resize and relabel as needed. You are building with Lego blocks, not drawing from scratch.

### Screen 1: Morning Briefing
This is the homepage. Think: what does the user read in 2 minutes each morning?

Place these blocks:
- **Top bar**: App name ("Mtn") + today's date
- **Hero card (large)**: Target adventure name + days until departure + readiness score (a number out of 100)
- **Text block**: Claude's briefing text (just a rectangle with "Today's briefing from Claude..." as placeholder)
- **Two small cards side by side**: Today's nutrition focus | Today's training note
- **Small link or button**: "Log today's training"

### Screen 2: Log Training Session
A simple form. One column, top to bottom:

Place these blocks:
- **Heading**: "Log Training Session"
- **Date field**
- **Activity type dropdown**: (Hiking / Running / Cycling / Strength / Other)
- **Duration field**: hours and minutes
- **Intensity selector**: Easy / Moderate / Hard (three buttons)
- **Notes text area**
- **Submit button**: "Save Session"

### Screen 3: Set Target Adventure
A form + summary panel:

Place these blocks:
- **Heading**: "My Target Adventure"
- **Trail name field**
- **Start date field**
- **Duration field**: number of nights
- **Difficulty selector**: (Moderate / Hard / Extreme — three buttons)
- **Summary card** (right side or below): Shows trail name, dates, and an estimated "fitness requirement" as a placeholder number

### Screen 4: Meal Plan
A read-only view generated by Claude:

Place these blocks:
- **Heading**: "3-Day Meal Plan"
- **Subtitle**: "Optimized for kidney health · anti-inflammatory · cardiovascular"
- **Three columns** (Day 1, Day 2, Day 3), each containing:
  - Breakfast card
  - Lunch card
  - Dinner card
- **Regenerate button** at the bottom: "Generate New Plan"

---

## Step 4 — Gut-Check Each Screen (5 min)

For each of the 4 screens, ask yourself one question:

> *"If I opened this screen, would I immediately know what to do?"*

If yes — move on.
If no — something is missing or buried. Make the primary action (button, number, text block) bigger or more obvious. Don't overthink it.

---

## Step 5 — Get the Figma URL and Bring It to Claude

1. In Figma, click **Share** (top right)
2. Set access to **"Anyone with the link can view"**
3. Copy the link — it will look like:
   `https://www.figma.com/design/XXXXXXXX/Mtn-Wireframes?node-id=0-1`
4. Paste the URL into Claude Code and say:
   > "Analyze these wireframes. Tell me: what components do I need, what data does each component require, and what are the UX gaps?"

Claude will produce a component inventory, data shapes, and a build order. That output becomes the technical brief for Cycle 0.

---

## What "Good Enough" Looks Like

You don't need polish. You need:
- [ ] All 4 screens exist
- [ ] Every screen has a clear primary element (the thing your eye goes to first)
- [ ] Forms have all their fields labeled
- [ ] Cards are labeled with what data they will show

That's it. If you have those four things, Claude can work with it.
